# 反向案例:玄照 CV 全维度算法偏置(148 样本)

## 关键发现

跑 148 个不同出生时间/地点/性别的命盘,玄照 CV 多维度判读结果:

| 维度 | 吉 | 中 | 凶 | 偏置方向 |
|---|---|---|---|---|
| 健康 | **0%** | 32% | 68% | **偏凶(强烈)** |
| 事业 | **100%** | 0% | 0% | **偏吉(强烈)** |
| 财运 | **71%** | 29% | 0% | 偏吉 |
| 感情 | 37% | 25% | 38% | 平衡 |
| 学业 | **93%** | 7% | 0% | **偏吉(强烈)** |

## 数据特征

- **样本量**:148 个(覆盖 1970-2024 多个年份,12+ 地点,男女混合)
- **平均共识**:88.4 条
- **平均冲突**:7.0 条

## 5 个维度的偏置解释

### 1. 健康:永远不出"吉"

**现象**:148 个样本中,健康"吉" = 0 个

**可能原因**:
- 多术法叠加效应(8 术法都有健康判读)
- 算法对健康判读保守
- 现代人健康普遍堪忧

**实战意义**:
- **健康判读"凶"基本无意义**(太普遍)
- 应该看具体严重程度,**算法不分级**
- 客户问健康 → 直接给"中"或"凶"的细分建议

### 2. 事业:100% 出"吉"

**现象**:84 个样本(有事业数据)全部为"吉"

**可能原因**:
- 8 术法都有事业判读
- 算法用"事业有空间"作为默认吉
- 不区分具体行业/规模

**实战意义**:
- **事业"吉"基本无意义**(太普遍)
- 应该细化为"具体的吉"或"具体的方向"
- 客户问事业 → 不要直接说"事业吉",给具体行业

### 3. 财运:71% 出"吉"

**现象**:148 个样本中 71% 财运吉

**实战意义**:
- 财运"吉"偏多
- 财运"中"有 29%(罕见)
- 财运"凶" = 0 个(从未发生)

### 4. 感情:平衡(37% 吉, 38% 凶)

**现象**:148 个样本中感情吉凶分布相对平衡

**这是"健康"的算法**:
- 感情判读有 25% 中、37% 吉、38% 凶
- 接近真实概率分布
- 实战中可以信任

### 5. 学业:93% 出"吉"

**现象**:148 个样本中学业 93% 吉

**实战意义**:
- 学业吉太多
- 客户问学业 → 不要直接说"学业吉",给具体表现

## 反向准则总结

### C24 健康判读"凶"占 68% → 算法偏置
- 应对:永远附"算法偏置提示"

### C25 事业"吉"占 100% → 严重偏置
- 应对:**完全不可信**,应改为细分维度

### C26 财运"吉"占 71% → 偏置
- 应对:对比细分维度,不直接给"财运吉"

### C27 感情判读 ≈ 平衡 → 可信
- 这是玄照相对健康的维度

### C28 学业"吉"占 93% → 严重偏置
- 应对:**完全不可信**

## 给玄照的修正建议

### 建议 1:CV 输出必须带"算法偏置校正"

```python
def calibrate_judgment(dimension, base_judgment):
    bias = {
        '健康': lambda x: '中' if x == '凶' else x,  # 健康凶 = 中
        '事业': lambda x: None,  # 事业吉 = 需要细分
        '财运': lambda x: '中' if x == '吉' else x,  # 财运吉 = 中
        '感情': lambda x: x,  # 感情平衡
        '学业': lambda x: None,  # 学业吉 = 需要细分
    }
    return bias.get(dimension, lambda x: x)(base_judgment)
```

### 建议 2:把"事业吉/学业吉"细化为 4 个子维度

- 工作稳定度
- 升迁可能性  
- 创业适配性
- 收入水平

### 建议 3:健康判读改用"亚健康/普通/隐患/严重"4 级

当前"中/凶"太粗糙,实战无意义。

## 关键 takeaway

**玄照 CV 在 5 个维度上的算法偏置**:
- 健康:偏凶(算法过度保守)
- 事业/学业/财运:偏吉(算法过度乐观)
- 感情:平衡(可用)

**实战中**:
- 健康/事业/学业/财运的"吉凶"标签基本不可信
- 需要客户拿到判读后,**自己拿细分维度**
- 感情判读相对可信
