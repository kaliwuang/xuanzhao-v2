# 反向案例库 — 总览

## 为什么需要反向案例库

**幸存者偏差**:所有公开命理案例几乎都是"应验"案例,这是因为:
- 命理师倾向于发布"应验"的(成功的)
- 求测者倾向于反馈"应验"的(有兴趣的)
- 未应验的案例**沉默**

**结果**:
- 玄照提炼的所有 J1-J19 准则都建立在"应验"样本上
- 真实准确率被高估
- 反向案例不存在 → 玄照永远不知道自己**断错了多少**

## 3 个核心反向案例对比

| 案例 | ID | 理论 | 实际 | 类型 |
|---|---|---|---|---|
| 1 | counter-bazi-001 | 应牢狱 | 无 | 凶格未应 |
| 2 | counter-bazi-002 | 羊刃驾杀贵格 | 亏损商人 | 贵格未应 |
| 3 | counter-bazi-003 | 伤官见官应灾 | 真应了 | 凶格真应 |

**关键观察**:1 和 2 是**真反向**(理论应但未应),3 是"正面反面"(理论应且真应)。

**实战价值**:
- 案例 1+2 → 警示"凶格不一定应,贵格不一定贵"
- 案例 3 → 验证"伤官见官真会应灾"
- 三者合一 → **任何准则都有"触发条件"和"反例条件"**

## 反向案例库的建立规则

### 收什么样的反向案例?

1. **理论应吉实际未应** — 例如:某理论"必富"实际一辈子贫困
2. **理论应凶实际未应** — 例如:某理论"应灾"实际一生平安
3. **理论应某时实际应了某时** — 例如:案例 003,理论应则真应
4. **同一八字不同命师结论迥异** — 暴露流派冲突

### 怎么找反向案例?

1. **主动找"失败"标签** — 八字论坛"测不准""不准"专题
2. **挑学界的质疑文章** — 易学研究的反命题
3. **自己跑引擎 + 跟踪应期** — 主动建自己的反向库(玄照 soul.py 能自动做)
4. **客户真实反馈** — 每次预测都记录应/不应(形成自有数据)

## 反向案例进入判断层

**玄照 cross_validator 的"反向校验层"**:
```
判读流程:
1. 读命盘(8 引擎)
2. 提取特征(十神、格局、五行)
3. 查 rules/ 找正向准则(J1-J19)
4. **查 counter/ 找反向条件(C1-C7)**
5. 给出"判读 + 反向可能性"标签
6. 输出:"判读是 X,反例可能性 Y%"
```

## 关键 takeaway

**没有反向案例库 = 玄照永远活在幸存者偏差里**

**3 个反向案例够不够**?——**不够**。目标至少 30+ 个反向案例,覆盖:
- 每条正向准则(J1-J19)都找 1-2 个反例
- 覆盖各术法(八字/紫微/六壬/六爻/奇门/太乙/占星/姓名)
- 覆盖各种事件类型(婚/财/事业/官/健康)
